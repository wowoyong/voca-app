## 현재 상태 분석

### ✅ 기술적 강점

- **라이선스 클린**: 모든 의존성이 MIT/Apache-2.0 (상업적 사용 가능)
- **PWA 아키텍처**: 앱스토어 수수료(30%) 없음, 배포 자유도 높음
- **SRS 알고리즘**: SM-2 기반 Spaced Repetition 구현
- **멀티 언어**: 영어/일본어 각 2,000단어
- **안정적 인증**: JWT + bcryptjs (프로덕션 준비 완료)

### ⚠️ 현재 제약사항

- **SQLite**: 동시접속 제한 (MAU 1천~수천까지만 권장)
- **콘텐츠 저작권 미확인**: 2,000단어 출처 검증 필요
- **법적 문서 부재**: 개인정보처리방침, 이용약관, 환불정책
- **자체 서버**: Mac Mini (확장성/안정성 리스크)
- **마케팅 채널 부재**: 사용자 유입 전략 필요

### 주요 기능

1. **오늘의 학습**: 매일 15개 새 단어 (seeded random)
2. **퀴즈**: 4지선다 (today/review 모드)
3. **플래시카드**: 빈칸 채우기 (today/review 모드)
4. **복습 시스템**: SM-2 알고리즘 기반 자동 스케줄링
5. **학습 통계**: 연속 학습일, 학습 단어 수, 복습 대기 단어
6. **TTS**: 영어/일본어 발음 재생 (24시간 캐싱)

## 비즈니스 모델 비교

### 1️⃣ 완전 광고 모델

**수익 구조**:
- Google AdSense 또는 카카오 애드핏
- 배너 광고 + 전면 광고

**장점**:
- ✅ 진입장벽 낮음 (결제 시스템 불필요)
- ✅ 전환율 걱정 없음
- ✅ 구현 비교적 간단

**단점**:
- ❌ 낮은 수익성 (MAU 10,000명 = 월 50~300만원)
- ❌ 사용자 경험 저하 (광고 피로도)
- ❌ 학습 집중도 방해 → 리텐션 하락
- ❌ 수익 예측 불안정
- ❌ 트래픽이 많아야 의미 있는 수익

**수익 예상**:
```
MAU 1,000명:   월 5~20만원 (서버비도 안 나옴)
MAU 10,000명:  월 50~300만원 (인프라 비용 커버 힘듦)
MAU 100,000명: 월 500~3,000만원 (이 정도는 되어야 의미 있음)
```

## 카카오 로그인 구현 가이드

### 난이도: ⭐⭐☆☆☆ (중하)

**소요 시간**: 3~4시간

### 1단계: 카카오 개발자 등록 (30분)

1. https://developers.kakao.com 접속
2. 카카오 개발자 계정 가입
3. **내 애플리케이션** > **애플리케이션 추가하기**
4. **REST API 키** 발급
5. **플랫폼 설정** > **Web 플랫폼 등록**
   - 개발: `http://localhost:3005`
   - 프로덕션: `https://yourdomain.com`
6. **Redirect URI 등록**
   - `http://localhost:3005/api/auth/kakao/callback`

### 2단계: 환경 변수 추가 (5분)

```env
# .env
KAKAO_REST_API_KEY="your-rest-api-key-here"
KAKAO_REDIRECT_URI="http://localhost:3005/api/auth/kakao/callback"
```

### 3단계: DB 스키마 수정 (10분)

```prisma
// prisma/schema-auth.prisma
model WebUser {
  id           Int      @id @default(autoincrement())
  username     String
  passwordHash String?  // null 허용 (소셜 로그인)
  kakaoId      String?  @unique  // 추가
  createdAt    DateTime @default(now())
  updatedAt    DateTime @updatedAt
}
```

마이그레이션:
```bash
npx prisma migrate dev --schema=prisma/schema-auth.prisma --name add_kakao_id
npx prisma generate
```

### 4단계: API 라우트 구현 (2시간)

파일 구조:
```
src/app/api/auth/
├── kakao/
│   └── route.ts          # 카카오 로그인 시작
└── kakao/
    └── callback/
        └── route.ts      # 카카오 콜백 처리
```

구현 코드는 `/tmp/kakao-login-route.ts`, `/tmp/kakao-callback-route.ts` 참고

### 5단계: 로그인 페이지 수정 (30분)

```tsx
// src/app/login/page.tsx
<button 
  onClick={() => window.location.href = '/api/auth/kakao'}
  className="w-full bg-[#FEE500] text-black p-3 rounded-xl font-bold">
  카카오톡으로 3초 만에 시작
</button>
```
